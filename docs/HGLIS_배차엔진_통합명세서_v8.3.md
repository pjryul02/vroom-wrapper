## 1. 프로젝트 배경 및 목적

### 1.1 추진 배경

**현대리바트 가구 설치 물류의 현황**:

1. **가구 설치 물류의 특수성 및 복잡도 증가**
   - 단순 배송이 아닌 제품별 조립/설치 시간, 난이도 등 복합적인 제약조건 존재
   - 1인/2인 조립 구분, 기사별 기능도(Skill), 권역 등 세분화된 자원 관리 필요성 증대

2. **수기/경험 의존적 배차 프로세스 한계**
   - 기존 화주사 ERP 기반의 단순 수기 배차로 인해 관리자의 업무 부하 가중
   - 배차 담당자의 개인 역량 및 경험에 의존하여 기사 간 업무 편차 발생 및 최적 라우팅 도출 실패
   - 이동거리 및 설치 시간을 종합적으로 고려하지 못한 배차로 유류비 낭비 및 현장 대기 시간 발생

3. **실시간 모니터링 및 데이터 가시성 부재**
   - 현장 도착, 설치 시작/종료, 미결/반품 등 실시간 진행 단계에 대한 모니터링 체계 미흡
   - 기사 위치 및 설치 진행률에 대한 데이터 부재로 고객 문의(VOC)에 대한 즉각적인 응대 어려움

4. **시스템 연계성 부족 및 확장성 한계**
   - 현대리바트 ERP - HGLIS 간 데이터 인터페이스 미구축 → 정보 불일치 발생
   - 설치 기사를 위한 전용 App 기능 부족으로 현장 업무(상차, 설치, 결과 등록) 효율성 저하

### 1.2 프로젝트 목적

**2가지 핵심 목표**:

#### 목적 1: 배차 최적화 엔진 도입
- 위밋모빌리티의 루티(ROOUTY) 엔진 연동
- 시스템 기반 배차 & 실적 모니터링 체계화
- 현장 설치시간 & 이동시간 등을 고려한 최적 오더 분배
- 고객 관점 서비스 제공

#### 목적 2: 현장 실행력 강화
- 현장 친화적인 설치 기사 전용 하이브리드 App 구축
- 상차부터 설치 완료까지 전 과정을 실시간 데이터화
- 고객에게 서비스 리포트 즉시 발송
- 서비스 품질 획기적 개선

## 2. 시스템 개요

### 2.1 배차 목적

**고객 관점의 목적**:
- 희망시간, 약속한 시간대에 정확하게 방문
- 문제발생시 대응력 강화화
- 친절하고 전문적인 서비스

**운영 관점의 목적**:
- 기사별 효율적 분배와 통제
- 효율배차, 정확한 시간관리리
- 설치 시간 및 대기 시간 최적화
- 오더물량에 따른 효율적 배차차

### 2.2 핵심 제약조건 (8가지지)

| # | 제약조건 | 유형 | 설명 |
|---|----------|------|------|
| C1 | 2인품목-2인팀 매칭 | **하드** | 2인 필요 품목은 반드시 2인팀 또는 합배차 ||
| C2 | 기본설치비 하한 | 소프트 | 기사 일일 설치비 총합의 최소 보장선 | *일일합산필요*|
| C3 | 희망배송시간 | **하드** | 고객 지정 시간대 준수 ||
| C4 | 필요기능도 | **하드** | 제품별 필요 기능도 ≤ 기사 기능도 ||
| C5 | CBM | **하드** | 차량별 적재용량 한계 ||
| C6 | 월단위 설치비 상한 | 소프트 | 서비스등급별 월 상한 관리 | *일/월 누적합산 필요*|
| C7 | 신제품/신입 회피 | **하드** | 관리자 수동 체크, 신입 기사 신제품 배정 제외 ||
| C8 | 미결 이력 모델 회피 | **하드** | 과거 설치 미숙 미결 발생 모델 재배정 제외 ||

│  용도: 월상한 한도 결정 [소프트 제약]                        │
│                                                             │
│  S등급 → 월 상한 최고 (가장 많이 벌 수 있음)               │
│  A등급 → 월 상한 높음                                       │
│  B등급 → 월 상한 중간                                       │
│  C등급 → 월 상한 낮음 (제한적)                              │
│                                                             │
│  → 서비스등급이 높으면 배차 더 많이, 비싸게 받을 자격       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 8.3.1 거리비 계산 기준

**정의**: 상차지(물류센터)에서 가장 먼 배송지까지의 **실경로 거리**에 따른 거리비 (유류비 성격)

**산출 시점**: 배차 후 (후처리)

```
┌─────────────────────────────────────────────────────────────┐
│                    거리비 산출 기준                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ■ 거리 계산 기준 ⭐ 중요                                   │
│  ─────────────────────                                      │
│  - 기준점: 상차지 (물류센터/거점) ← 기사 거주지 아님        │
│  - 측정 대상: 상차지 → 가장 먼 배송지                       │
│  - 거리 산출: 도로 기준 실경로 거리 ← 유클리드 거리 아님    │
│                                                             │
│  ■ 기사 일일 수익                                           │
│  ─────────────────                                          │
│  기사 일일 수익 = 설치비 총합 + 거리비                      │
│                                                             │
│  ■ 수도권 (서울/경기/인천)                                  │
│  ─────────────────────────                                  │
│  │ 거리 (실경로)     │ 거리비      │                       │
│  ├───────────────────┼─────────────┤                       │
│  │ 30km 미만         │ ₩0          │                       │
│  │ 30~50km           │ ₩24,000     │                       │
│  │ 50~70km           │ ₩36,500     │                       │
│  │ 70~100km          │ ₩48,000     │                       │
│  │ 100~150km         │ ₩81,000     │                       │
│                                                             │
│  ■ 지방 (이외 지역)                                         │
│  ───────────────────                                        │
│  │ 거리 (실경로)     │ 거리비      │                       │
│  ├───────────────────┼─────────────┤                       │
│  │ 70km 미만         │ ₩0          │                       │
│  │ 70~100km          │ ₩40,500     │                       │
│  │ 100~150km         │ ₩81,000     │                       │
│  │ 150~200km         │ ₩93,000     │                       │
│  │ 200km 이상        │ ₩118,000    │                       │
│                                                             │
│  ■ 예시                                                     │
│  ────────                                                   │
│  기사 A (수도권):                                           │
│    상차지: 용인 물류센터                                    │
│    배송지: 오더1(15km), 오더2(42km), 오더3(28km)            │
│    최장거리: 42km (실경로) → 거리비: ₩24,000                │
│    설치비 총합: ₩280,000                                    │
│    ─────────────────────────────────                        │
│    기사 일일 수익: ₩280,000 + ₩24,000 = ₩304,000            │
│                                                             │
│  기사 B (지방):                                             │
│    상차지: 대전 물류센터                                    │
│    배송지: 오더1(85km), 오더2(120km), 오더3(65km)           │
│    최장거리: 120km (실경로) → 거리비: ₩81,000               │
│    설치비 총합: ₩350,000                                    │
│    ─────────────────────────────────                        │
│    기사 일일 수익: ₩350,000 + ₩81,000 = ₩431,000            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```

│  ■ C7 vs C8 차이                                            │
│  ────────────────                                           │
│  C7: 신입이라 경험 없음 → 신제품 전체 회피                  │

│                                                             │
│  ■ 필요 데이터                                              │
│  ─────────────                                              │
│  기사 마스터에 "회피모델리스트" 추가                        │
│  - 최소 정보: 모델코드 + 미결발생일시                       │
│                                                             │
│  예시:                                                      │
│  {                                                          │
│    "driver_id": "D001",                                     │
│    "avoid_models": [                                        │
│      { "model_code": "BED-A01", "date": "2025-01-10" },     │

│    ]                                                        │
│  }                                                          │
│                                                             │
│  ■ 데이터 생성 흐름                                         │

│  1. 미결 히스토리 테이블에서 조회                           │
│     - 미결사유 = "설치미숙" 또는 유사 코드                  │
│  2. 해당 건의 모델코드 + 일시 추출                          │
│  3. 기사 마스터 또는 별도 테이블에 적재                     │
│  4. 배차 시 참조                                            │
│                                                             │
│  ■ 처리 전략                                                │
│  ─────────────                                              │
│  Option A: 배치 갱신 (권장)                                 │
│    - 매일 새벽 미결 히스토리 스캔 → 회피모델 리스트 갱신    │

│    - 즉시 반영됨                                            │
│                                                             │
│  ■ 배차 적용                                                │
│  ─────────────                                              │
│  - 오더의 모델코드 ∈ 기사의 회피모델리스트 → 배정 제외     │
│  - skill 매핑 또는 전처리 필터링으로 처리                   │

│  ───────────────                                            │
│  - 회피모델 리스트 조회/관리 화면 필요                      │

│                                                             │
└─────────────────────────────────────────────────────────────┘

│  ─────────────                                              │

### 9.4 합배차 물리적 매칭 분석

**전처리에서 수행하는 사전 검증**:

```python
for order in 2인_오더_미배정:
    
    # 1단계: 기능도 충족 기사 필터링
    # 특례: 둘 중 1명만 기능도 충족되어도 OK
    candidate_drivers = [
        d for d in joint_dispatch_pool
        if (d.기능도 >= order.필요기능도) OR
           (any_partner_has_skill(d, order.필요기능도))
    ]
    
    # 2단계: 2명 조합 생성
    for driver_a, driver_b in combinations(candidate_drivers, 2):
        
        # 3단계: 동일 시간 블록 확인
        common_time = find_common_time_block(driver_a, driver_b, order)
        if not common_time:
            continue
        
        # 4단계: CBM 검증 (둘 다 적재 가능)
        if (driver_a.가용CBM < order.CBM or 
            driver_b.가용CBM < order.CBM):
            continue
        
        # 5단계: 거리 효율 (둘 다 50km 이내)
        dist_a = calculate_distance(driver_a.location, order.location)
        dist_b = calculate_distance(driver_b.location, order.location)
        
        if dist_a > 50 or dist_b > 50:  # km
            continue
        
        # 매칭 성공 → job 복제 생성
        create_joint_jobs(order, driver_a, driver_b, common_time)
        break
```
